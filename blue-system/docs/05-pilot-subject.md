# 05 — Picking a Pilot Subject

Trying to be "Wikipedia for doing things" on day one is how this dies. Pick one subject and prove the model.

## Criteria for a good pilot
1. **Enthusiastic existing community** willing to write for free.
2. **Tangible outputs** so guides can be tested by following them.
3. **Hierarchy comes naturally** (clear progression from basics to expert).
4. **Low legal risk** (not medicine, not firearms, not drugs).
5. **Topical product links exist** (so monetization can be tested later).

| **Medicine** | Strong | Yes | Clear | **HIGH** | Yes | NO — phase 4 at earliest |

## Recommendation
**Start with home electronics.** It has:
- A community that already writes detailed build logs.
- Perfectly clean hierarchy (transistor → logic gate → ALU → CPU).
- Sebastian literally used it as his example in the video.
- Parts vendors (Mouser, Digi-Key, Adafruit, etc.) for the eventual ad model.

## What "success" looks like at end of pilot
- 200+ Level 1–2 guides.
- 50+ Level 3+ guides.
- 30+ active verifiers.
- 5+ resolved disputes (you want to see the system stress-tested).
- ~10,000 unique monthly readers.
- Zero takedown notices honored against canonical guides.
