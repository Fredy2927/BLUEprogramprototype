# B.L.U.E. — Broad Learning Universal Education

> A free, community-built "how to do anything" website.
> Originally proposed by **Sebastian Rey** ([video](https://www.youtube.com/watch?v=qcRKmm3B25c),
> [forum](https://github.com/SebReyWriter/TheBLUEForum/discussions)).
> This repo is a starter kit for someone "picking up the tab."

---

## What's in this folder

| File | What it is |
|---|---|
| `docs/00-vision.md` | One-page mission + non-negotiables (kept faithful to Sebastian's video) |
| `docs/01-roadmap.md` | Phased plan: Spec → MVP → Pilot subject → Open beta |
| `docs/02-spec.md` | Full system spec: hierarchy, methods, verifiers, disputes, monetization |
| `docs/03-data-model.md` | Database schema sketch (entities + relationships) |
| `docs/04-legal-and-jurisdiction.md` | Risks Sebastian raised + concrete first steps |
| `docs/05-pilot-subject.md` | Why to start with ONE subject as proof-of-concept |
| `docs/06-outreach-to-sebastian.md` | Draft message you can send him |
| `prototype/index.html` | A self-contained, no-build HTML demo of the hierarchy + guide UI |

---

## Suggested next moves (in order)

1. **Read** `docs/00-vision.md` and `docs/01-roadmap.md`. Decide if you agree.
2. **Open** `prototype/index.html` in a browser — this is the look/feel to react to.
3. **Reach out** to Sebastian using `docs/06-outreach-to-sebastian.md`. He explicitly asked for someone to pick it up; getting his blessing (or even his involvement) is huge for legitimacy.
4. **Pick a pilot subject** (`docs/05-pilot-subject.md`) — don't try to cover "everything" first. Prove the model on one niche.
5. **Choose a stack** and start scaffolding the real app. (I can help you pick and build this — just say the word.)
